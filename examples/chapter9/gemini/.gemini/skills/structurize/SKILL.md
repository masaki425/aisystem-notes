# 論文構造化スキル

input/ にある論文PDFを読み、以下の手順で概念ネットワークを抽出してYAMLに出力する。
**手順を飛ばさず、順番通りに実行すること。**

## 手順

### Step 1: 論文の概要把握

論文全体を読み、以下を特定する:
- 論文の主要な主張（1文で）
- 研究の手法（何をどうやって調べたか）
- 主要な結果（何がわかったか）
- 理論的意義（何に貢献するか）

結果を `## 概要把握` としてYAMLファイルのコメントに記録してから次に進む。

### Step 2: ノード候補の抽出

論文から概念ネットワークのノードとなる要素を抽出する。以下の優先順で探す:

1. **論文の中心対象**: 論文が発見・提案・検証した主要な対象（分子、理論、手法等）
2. **直接の関連要素**: 中心対象が作用する対象、必要とする条件、生成する結果
3. **比較対象**: 中心対象と比較されている既存の概念・手法
4. **理論的枠組み**: 結果が支持・反証する理論や仮説
5. **課題・障害**: 中心対象が解決する問題、残された課題

各ノードに id (snake_case), label, type, description を付与する。

### Step 3: エッジの特定

ノード間の関係を特定する。以下の観点で探す:

1. **因果関係**: AがBを引き起こす（causes）、AがBを生成する（produces）
2. **依存関係**: BがAを必要とする（requires）
3. **支持関係**: AがBの証拠となる（supports）
4. **阻害関係**: AがBを妨げる（inhibits）
5. **構成関係**: AがBを含む（contains）
6. **比較関係**: AとBが比較される（compares）
7. **派生関係**: AからBが派生する（derives）

各エッジに source, target, relation, description を付与する。

### Step 4: 整合性の自己チェック

YAML出力前に以下を確認する:
- 全エッジのsource/targetがnodesのIDに存在するか
- 孤立ノード（どのエッジにも参照されないノード）がないか。あれば削除するか、エッジを追加する
- 同じ関係の重複エッジがないか

### Step 5: YAML出力

output/ にYAMLファイルを出力する。
