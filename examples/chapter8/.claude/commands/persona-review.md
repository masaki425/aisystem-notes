ペルソナエージェント群が対象成果物を独立してレビューし、
「考えたか？」という問いを生成する。
全フェーズの実行と統合レポートの生成まで完了しない。

★★ このコマンドが変更して良いファイルは logs/persona_review.md と proposal.md（Fセクションのみ）。★★
spec.md、src/、scripts/ 等のプロジェクトファイルを Write / Edit しない。
問いを生成してレポートに記録するだけ。修正はユーザーが判断する。

【引数】

フェーズをカンマ区切りで指定する。省略時は全フェーズ。

  academic    proposal.md を対象（研究視点）
  design      proposal.md + spec.md を対象（設計視点）
  impl        spec.md + scripts/ + output/ を対象（実装視点）

  省略時: 全フェーズを順次実行（academic → design → impl）
  対象成果物が存在しないフェーズは自動スキップ

【実行手順】

1. 実行フェーズを決定する（引数 or デフォルト全フェーズ）
2. 各フェーズの対象成果物が存在するか確認
3. フェーズごとに順次実行:
   a. 対象成果物を読み込む
   b. Task ツールで該当フェーズの全ペルソナを同時起動
   c. 全 Worker の完了を待機
   d. 問いを収集
4. 全フェーズ完了後、全問いを統合して収束分析を行う
5. logs/persona_review.md に統合レポートを生成する

【完了報告】

「ペルソナレビュー完了。
実行フェーズ: {実行したフェーズ}
問い: {N}件（収束: {M}件）

logs/persona_review.md を確認してください。」
