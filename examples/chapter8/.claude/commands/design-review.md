spec.md の設計をレビューし、問題の程度を評価する。
レビュアーリストの全員を実行し、統合レポートまで出力して完了する。

★★ このコマンドは読み取り専用。ファイルの変更・修正は一切行わない。★★
spec.md、src/、scripts/ 等のプロジェクトファイルを Write / Edit しない。
問題を発見したら報告するだけ。修正はユーザーが /execute で別途実行する。

【引数】
レビュアーをカンマ区切りで指定。省略時は全レビュアー。
  self      ホストツール自身（Claude Code）
  codex     Codex CLI
  gemini    Gemini CLI

  省略時: 全レビュアーを順次実行（self → codex → gemini）
  利用不可なレビュアーは自動スキップ（CLI が見つからない場合）

【実行手順】
1. レビュアーリストを決定する
2. docs/spec.md を読む。既存コードがあれば src/ も確認
3. リストの先頭から順に実行する:

   **self の場合:**
   以下の観点でレビューし、結果を記録する。
   - 目的との適合性（達成可能か、YAGNI、不足）
   - 設計の一貫性（データ構造の矛盾、命名規則、責務分離）
   - 暗黙の前提（文書化されていない仮定、外部依存リスク）
   - 自動検証の妥当性（検証内容が成功基準をカバーしているか）
   - 並列化設定の妥当性

   **codex の場合:**
   `which codex` で存在確認。なければスキップ。
   `codex exec --sandbox read-only --cd $(pwd) "{PROMPT}"` を実行する。

   **gemini の場合:**
   `which gemini` で存在確認。なければスキップ。
   `gemini -p "{PROMPT}"` を実行する。

4. 全レビュアー完了後、統合レポートを出力する

【無視して良い指摘】
- 「要件定義書がない」「requirements.md がない」などの指摘
