実装結果を確認し、問題の分析と修正提案を行う。
レビュアーリストの全員を実行し、統合レポートまで出力して完了する。

★★ このコマンドが変更して良いファイルは logs/issues.md のみ。★★
spec.md、src/、scripts/ 等のプロジェクトファイルを Write / Edit しない。
問題を発見したら issues.md に記録して報告するだけ。修正はユーザーが /execute で別途実行する。

【引数】
レビュアーをカンマ区切りで指定。省略時は全レビュアー。
  self      ホストツール自身（Claude Code）
  codex     Codex CLI
  gemini    Gemini CLI

  省略時: 全レビュアーを順次実行（self → codex → gemini）
  利用不可なレビュアーは自動スキップ（CLI が見つからない場合）

【実行手順】
1. レビュアーリストを決定する
2. logs/issues.md と docs/spec.md を読む。src/ と output/ も確認
3. リストの先頭から順に実行する:

   **self の場合:**
   issues.md の未解決問題を読み、以下の観点でレビュー。
   - バグ・エラーハンドリング
   - 仕様との整合性（spec.md の成功基準との照合）
   - コード品質
   - データ品質（空フィールド、異常値、フィールド名の整合性）

   **codex / gemini の場合:**
   存在確認後、外部レビュー用共通プロンプトで実行。

4. 全レビュアー完了後、統合レポートを出力する

【無視して良い指摘】
- 「要件定義書がない」「requirements.md がない」などの指摘
