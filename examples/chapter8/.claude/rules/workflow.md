# ワークフロールール

## 実行原則
- 「次にやること」に書いていない作業をしない
- 手順をスキップしない
- spec.md にない機能を追加しない
- 自己判断で目的・成功基準を変更しない
- 完了済みの Task を再実行しない
- progress.md に書いていない状態を仮定しない
- 「プロジェクトの目的」から逸脱した実装をしない
- 目的に貢献しない余計な作業をしない

## ファイル変更後の git commit 義務
プロジェクトファイルを Write / Edit した場合、作業の区切りで必ず git add + git commit する。
コマンド経由（/execute, /result-review 等）でも自由文指示でも同じ。
Stop hook が未 commit を検出するが、hook に頼らず能動的に commit すること。

## spec.md 変更時の記録義務
spec.md を変更（Write/Edit）した場合は、必ず以下を docs/progress.md の作業ログに記録すること:
- 変更箇所（どのセクション・どの項目を変えたか）
- 変更理由（なぜ変えたか）
- 変更前後の差分の要約

## コンテキスト管理
- 大きなファイル（1MB超）は構造確認後に必要部分だけ取得
- 不要な情報をコンテキストに入れない
- 外部 API 呼び出しは最小限に

## Stop hook 失敗時の対応
Stop hook が {"ok": false} を返した場合、reason に記載された問題と対処法をユーザーに報告する。

## 予期せぬ終了からの復帰
API エラーやクラッシュで強制終了した場合：
1. docs/progress.md の「次にやること」と「作業ログ」を確認
2. 最後に成功した地点から再開
3. 不整合があれば報告してから継続
