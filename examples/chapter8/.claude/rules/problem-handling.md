# 問題発見時の対応

## 判断基準
「この問題があっても、現在のタスクを継続できるか？」で判断する。

## 継続可能な問題（優先度: 低/中）
対応:
1. logs/issues.md の「未解決問題サマリー」テーブルに追加
2. logs/issues.md の「詳細」セクションに詳細を記録
3. 現在のタスクを継続（中断しない）

## 継続不可の問題（優先度: 高）
対応:
1. progress.md を更新（status: blocked、blocked_by に概要）
2. logs/issues.md に記録
3. git commit（"中断: 問題を発見"）
4. 作業を中断して報告

## 問題を解決した場合
1. issues.md のサマリーテーブルから該当行を削除
2. 詳細セクションの「対応状況」を「対応済み」に変更
