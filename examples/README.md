# 実践例：段階的にHarnessを構築する

## シナリオ

毎週のメモ（箇条書き）から、構造化された週次レポートを生成する。
同じタスクに対して、Harnessの機能を1つずつ追加していくことで、
各機能が何を解決するかを体感する。

## 4つのステップ

| ステップ | 追加する機能 | 制御の種類 | 解決する問題 |
|---------|-------------|-----------|-------------|
| 01 | CLAUDE.md のみ | 確率的 | プロジェクトの前提をAIに伝える |
| 02 | + Rules | 確率的（強化） | 行動規則・禁止事項を定義する |
| 03 | + Skill | 確率的（再利用） | 手順を再利用可能にする |
| 04 | + Hook | 決定論的 | 必須条件を確実に守らせる |

## 試し方

各ディレクトリにClaude Codeで `cd` して作業する。
input/ にサンプルデータが入っているので、すぐに試せる。

```bash
# ステップ1を試す
cd examples/01_claude_md_only
# Claude Codeで「週次レポートを作って」と指示

# ステップ2を試す（Rulesを追加）
cd examples/02_with_rules
# 同じ指示を出して、振る舞いの違いを観察

# ステップ3を試す（Skillを追加）
cd examples/03_with_skill
# /report と入力するだけ

# ステップ4を試す（Hookを追加）
cd examples/04_with_hook
# /report と入力。Hookが品質を保証する
```

## 観察のポイント

- 01→02: Rulesを追加すると、AIの振る舞いは変わるか？ 常に守られるか？
- 02→03: Skillを使うと、毎回同じ品質の手順が実行されるか？
- 03→04: Hookを追加すると、何が「確実に」保証されるようになるか？
